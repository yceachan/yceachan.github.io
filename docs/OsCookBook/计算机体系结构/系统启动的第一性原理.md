# 系统启动的第一性原理：信任链与资源释放

在操作系统（Linux/Windows）接管世界之前，硬件从上电复位到加载内核的过程，是一个极其精妙的**接力赛 (Relay Race)**。
系统启动的第一性原理是：**链式信任 (Chain of Trust) 与 逐步解锁 (Incremental Unlock)。**

---

## 1. 创世纪：Reset Vector (复位向量)

CPU 上电后，所有的寄存器都是随机值，只有 PC (Program Counter) 被强制指向一个固定的物理地址（如 `0x00000000` 或 `0xFFFF0000`）。
*   **第一原则：** 此时只有 **ROM (Read-Only Memory)** 是可用的。DDR 还是一片死寂，连初始化都没做。
*   **代码：** 这里烧录的是 **BootROM**（通常由芯片厂商固化，不可修改）。

### 1.1 BootROM 的使命
1.  **极简环境：** 关闭所有中断，关闭 Cache，关闭 MMU。
2.  **寻找载体：** 扫描启动设备（SD卡、eMMC、SPI Flash）。
3.  **加载下一棒：** 将 **SPL (Secondary Program Loader)** 读取到片内的 **SRAM** 中。
    *   *为什么是 SRAM？* 因为 DDR 需要几百个寄存器的配置才能跑起来，BootROM 太小塞不下 DDR 驱动。

---

## 2. 第一棒：SPL / FSBL (SRAM 阶段)

SPL (Secondary Program Loader) 是 BootROM 拉起的第一段用户代码（通常是 U-Boot 的一部分）。
*   **第一原则：** **资源受限 (SRAM Constrained)。** 只有几十 KB 的空间。
*   **核心任务：** **初始化 DDR 控制器。**
    *   配置时序参数 (Timing Parameters)。
    *   进行 DDR 训练 (Training/Calibration)。
*   **交接：** DDR 初始化完成后，内存空间瞬间从 KB 级膨胀到 GB 级。SPL 将完整的 **U-Boot** 加载到 **DDR** 中，并跳转。

---

## 3. 第二棒：U-Boot (DDR 阶段)

U-Boot (Universal Bootloader) 是一个微型的裸机操作系统。
*   **第一原则：** **外设驱动 (Peripheral Drivers)。**
    *   它拥有网卡、USB、屏幕的驱动。
    *   它提供交互式的命令行 (Shell)。
*   **核心任务：** **加载内核 (Load Kernel)。**
    *   从文件系统 (Ext4/FAT) 读取 `zImage` 和设备树 `dtb`。
    *   设置启动参数 (`bootargs`)。
*   **交接：** 关闭 DMA，关闭中断，清理 Cache，将 CPU 的控制权移交给 Linux 内核入口。

---

## 4. 终局：Kernel (虚拟化阶段)

Linux 内核接管硬件后，做的第一件事就是 **开启 MMU**。
*   **第一原则：** **从实模式切换到保护模式。**
*   **核心任务：**
    1.  **自解压：** `zImage` 解压成 `vmlinux`。
    2.  **初始化子系统：** 内存管理、调度器、VFS。
    3.  **挂载根文件系统 (RootFS)。**
    4.  **启动第一个用户进程：** `/sbin/init` (PID=1)。

---

## 总结：信任链的传递

| 阶段 | 存储介质 | 核心能力 | 信任来源 |
| :--- | :--- | :--- | :--- |
| **BootROM** | On-chip ROM | **不可变** (Immutable) | 硅片本身 (Root of Trust) |
| **SPL** | On-chip SRAM | **初始化 DDR** | 签名校验 (Secure Boot) |
| **U-Boot** | External DDR | **文件系统/网络** | SPL 校验 |
| **Kernel** | External DDR | **虚拟内存/多任务** | U-Boot 校验 |
| **Android** | Flash/Storage | **应用程序** | Kernel 校验 (dm-verity) |

**启动过程 = 硬件资源从 0 到 100 的逐步释放，同时也是软件权限从 Ring 0 到 Ring 3 的逐步收缩。**
