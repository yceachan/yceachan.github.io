# 学习路线规划

## 当前笔记体系（已完成）

```
note/
├── SysCall/          系统调用全景、sys_call_table、页表/MMU、架构横向对比
├── 虚拟化/
│   ├── 程序和进程/   进程起源、task_struct、fork/COW、生命周期、进程组/会话、TTY/PTS
│   ├── 进程地址空间/ 虚拟内存布局、procfs、ASLR
│   └── 进程通信IPC/
│       └── signal/   信号概念、API矩阵、内核分流机制
└── BSP-Dev/
    └── LCD-Touch/    MIPI-DSI
```

---

## 推荐学习顺序

### 阶段一：IPC 体系补全（与 Signal 并列）

Signal 已完成，IPC 家族还缺：

```
进程通信IPC/
├── signal/        ✅ 已完成
├── pipe-fifo/     ← 最简单，管道原理与 fd 继承
├── shared-memory/ ← mmap 匿名/文件映射，与地址空间笔记直接衔接
├── semaphore/     ← P/V 操作、内核实现
└── unix-socket/   ← 本地高性能 IPC 终极形态
```

> 衔接性最强，地址空间 + 进程生命周期笔记已铺垫好基础。

---

### 阶段二：中断子系统（BSP 开发核心）

GPIO / MIPI-DSI BSP 笔记已有，中断是驱动开发的核心链路：

```
中断/
├── 00-硬件中断全景（IRQ、GIC）
├── 01-软中断与 tasklet
├── 02-workqueue（进程上下文中的延迟处理）
└── 03-中断与驱动开发实战（iMX6ULL 硬件挂钩）
```

> 对 BSP 方向价值最高，直接服务于 iMX6ULL 驱动开发。

---

### 阶段三：调度器（task_struct 的下一章）

task_struct 解剖已完成，自然引出：

```
调度器/
├── 00-CFS 完全公平调度器（红黑树、vruntime）
├── 01-实时调度策略（SCHED_FIFO / SCHED_RR）
└── 02-嵌入式实时性问题与优化
```

> 理论深度的自然终点，也与嵌入式实时性直接相关。

---

### 备选方向：VFS 与文件描述符

exec/fork 的 fd 继承逻辑的深层支撑：

- `fd → file → inode` 三元组
- `open/read/write` 内核路径
- 与 procfs 的底层联系（地址空间笔记已有使用）

---

## 优先级总结

| 优先级 | 方向 | 理由 |
|--------|------|------|
| ★★★ | IPC 补全 | 快速收尾，与现有笔记衔接最紧 |
| ★★★ | 中断子系统 | BSP 实践价值最直接 |
| ★★☆ | 调度器 | 内核理论深度延伸 |
| ★☆☆ | VFS | 基础但相对独立，可按需补充 |
