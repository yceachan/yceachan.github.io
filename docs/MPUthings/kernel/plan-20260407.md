# kernel 笔记库治理计划（2026-04-07）

> **Status (2026-04-07 收尾)**：Step 1–5 全部完成。`context/` 单 overview 已涵盖原 01/02/03 拆分内容，不再细分；`defer/07-kthread-pattern.md` 与 `sched/04-kthread-kworker.md` 合并落在 sched/。顶层 [`00-execution-model.md`](./00-execution-model.md) 已建立，`_legacy-schedule-overview.md` 已删除。

> [!note]
> **Ref:** 现状目录 `note/kernel/{BottomHalf,schedule}/`，对话日期 2026-04-07

## 1. 现状诊断

```
kernel/
├── BottomHalf/   00-overview, 01-softirq, 02-tasklet, 03-workqueue, 04-threaded-irq
└── schedule/     00-overview, 01-wait-queue, 02-softtimer, 03-workqueue(redirect),
                  04-sync-cheatsheet, 05-context-landscape, 06-cortexM-pendsv,
                  07-imx6ull-gpt-jiffies
```

核心问题：

1. **workqueue 双写**：`BottomHalf/03` 是正本，`schedule/03` 已是 redirect 占位 —— 重定向是临时方案，应在新结构里彻底消除。
2. **`schedule/` 名不副实**：当前内容是"等待队列 + 定时器 + 同步 + 上下文 + 硬件计时器"的杂烩，并非进程调度器（CFS / sched_class / load balance 缺席）。
3. **缺少分层**：机制（mechanism）、上下文（context）、时间基座（HW timer/tick）、同步原语（sync）平铺在同级。
4. **知识缺口**：调度器主体、tick/hrtimer/clockevents、RCU、per-cpu、kthread、completion、preempt/irq 关闭语义等。
