# 终端调用 vs execv：揭秘两种启动方式的底层差异

当我们执行一个编译好的 C 程序（如 `my_viewer`）时，可以通过两种常见方式：
1. **终端直接调用:** 在 Shell (如 Bash/Zsh) 中输入 `./my_viewer`
2. **代码级调用:** 在另一个 C 程序中使用 `execv("./my_viewer", args)`

这两种方式在表面上看似不同，但**在内核态，它们最终都殊途同归于 `execve` 系统调用**。它们的差异，完全在于**是谁在调用 `execve`，以及调用时传入了什么参数**。

## 实战现象对比

假设 `my_viewer` 是一个打印 `argc` 和 `argv` 的程序。

### 现象 A: 代码级调用 (通过 caller.c 中的 execv)
```text
[Parent says]: Child PID created: 65691. Now I will execv...
[Child Process: my_viewer]
I am running! My PID is 65691

--- Arguments Received (argc = 3) ---
argv[0]: BUSYBOX_ALIAS_TEST
argv[1]: arg_from_parent_1
argv[2]: arg_from_parent_2
```

### 现象 B: 终端直接调用 (./my_viewer)
```text
[Child Process: my_viewer]
I am running! My PID is 65751

--- Arguments Received (argc = 1) ---
argv[0]: ./my_viewer
```

## 底层差异深度解析

### 1. 谁是真正的“父进程”？

- **终端调用:** 当你在终端敲下 `./my_viewer` 并回车时，**Shell (Zsh 或 Bash)** 就是父进程。Shell 会调用 `fork()` 产生一个子进程，然后在子进程中调用 `execve`。
- **execv 调用:** 父进程是你自己编写的 `caller` 程序。

### 2. argv[0] 是怎么被决定的？

这是差异最明显的地方。

- **终端调用 (Shell 的行为):** 
  按照 Unix 的通用约定，Shell 作为一个标准且严谨的“启动器”，它在调用 `execve` 时，会**忠实地将你输入的命令字符串作为 `argv[0]` 传递下去**。
  你输入的是 `./my_viewer`，Shell 传给 `execve` 的 `argv[0]` 就是 `"./my_viewer"`。如果你将它加到环境变量并输入 `my_viewer`，`argv[0]` 就是 `"my_viewer"`。

- **execv 调用 (代码的行为):**
  在 `caller.c` 中，我们**手动构造**了参数数组：
  ```c
  char *args[] = {
      "BUSYBOX_ALIAS_TEST", // 我们强行捏造的 argv[0]
      "arg_from_parent_1",
      "arg_from_parent_2",
      NULL
  };
  execv("./my_viewer", args);
  ```
  在这里，内核不会去管 `my_viewer` 的真实文件名叫什么，它只会**无脑地把你提供的 `args` 数组拷贝到新程序的栈上**。因此，`my_viewer` 醒来后看到的 `argv[0]` 就是我们捏造的 `"BUSYBOX_ALIAS_TEST"`。

### 3. argc 为什么不同？

- **终端调用:** 你没有输入任何额外的参数，因此 Shell 构造的数组只有 `{"./my_viewer", NULL}`。内核解析出 `argc = 1`。
- **execv 调用:** 我们手动传入了 3 个非空字符串，内核解析出 `argc = 3`。

## 核心结论：Shell 只是一个“高级的 caller”

很多初学者认为 `./` 运行程序是某种特殊的系统机制。实际上：

**Shell (如 Bash, Zsh) 本质上就是一个一直在运行的 `while(1)` 循环。它不断读取你的输入，解析字符串，然后执行与我们 `caller.c` 完全相同的 `fork()` + `execve()` 逻辑。**

唯一不同的是，Shell 遵循了标准的 Unix 约定（将命令本身作为 `argv[0]`），而我们自己在写代码调用 `execv` 时，拥有**绝对的自由**，可以随心所欲地伪造任何参数。这种自由正是类似 BusyBox 这种“多面手”工具得以存在的基础。
