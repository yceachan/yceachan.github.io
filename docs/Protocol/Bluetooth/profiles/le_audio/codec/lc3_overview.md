# LC3 编解码器概览 (Low Complexity Communication Codec)

> [!note]
> **Ref:** [Docs/LE-Audio/LC3_v1.0.1.pdf](Docs/LE-Audio/LC3_v1.0.1.pdf)

LC3 是蓝牙 LE Audio 强制要求的音频编解码器。它是一种基于块的变换编解码器 (Block-based transform audio codec)，具有低算法延迟、低复杂度实现以及极宽的可调比特率范围。

---

## 1. 核心特性 (Key Features)

| 特性 | 支持范围 |
| :--- | :--- |
| **帧时长 (Frame Duration)** | 10 ms (及 7.5 ms) |
| **支持采样率 (Sampling Rates)** | 8, 16, 24, 32, 44.1, 48 kHz |
| **支持位深 (Bit Depth)** | 16, 24, 32-bit PCM |
| **比特率范围 (Bitrate)** | 16 kbps - 320 kbps (10ms 帧) / 21.3 kbps - 426.7 kbps (7.5ms 帧) |
| **单帧载荷 (Payload Size)** | 20 字节 - 400 字节 |

---

## 2. 延迟参数 (Algorithmic Delay)

LC3 的总算法延迟由帧时长和 MDCT（改进型离散余弦变换）前看 (Look-ahead) 延迟组成。

| 采样率 (kHz) | 帧时长 (ms) | 前看延迟 (ms) | 总算法延迟 (ms) |
| :--- | :--- | :--- | :--- |
| **8/16/24/32/48** | 10.0 | 2.5 | **12.5** |
| **8/16/24/32/48** | 7.5 | 4.0 | **11.5** |
| **44.1** | 10.88 | 2.72 | **13.6** |
| **44.1** | 8.16 | 4.35 | **12.52** |

> **注**: 对于 44.1 kHz 采样率，LC3 使用与 48 kHz 相同的帧大小（480/360 采样点），导致实际帧时长略长（480/441 倍）。

---

## 3. 编解码接口与配置 (Interfaces)

### 3.1 编码器输入 (Encoder Inputs)
- **采样率 $F_s$**: 8, 16, 24, 32, 44.1, 48 kHz。
- **帧时长 $N_{ms}$**: 7.5 ms 或 10 ms。
- **音频通道数 $N_c$**: 支持多通道编码（通过多个单声道流实现）。
- **位深**: 16, 24, 32-bit。
- **Byte Count**: 每帧目标载荷大小（20-400 字节），决定了压缩比特率。

### 3.2 解码器输入与异常处理 (Decoder Inputs & BFI)
- **BFI (Bad Frame Indication)**: 由链路层或应用层提供的外部标志。
    - `0`: 载荷正常。
    - `1`: 载荷损坏或丢失。
- **PLC (Packet Loss Concealment)**: 当 BFI=1 时，解码器会激活丢包补偿算法，利用历史帧预测输出 PCM。

---

## 4. 采样率索引 (Sampling Rate Index)

LC3 使用采样率索引 $f_{ind}$ 进行参数映射：

| $F_s$ (Hz) | 8,000 | 16,000 | 24,000 | 32,000 | 44,100/48,000 |
| :--- | :---: | :---: | :---: | :---: | :---: |
| **$f_{ind}$** | 0 | 1 | 2 | 3 | 4 |

---

## 5. 多通道支持 (Multi-channel)
LC3 算法本身描述的是**单通道**编码。任何双声道 (Stereo) 或多通道音频均通过编码多个单声道流来支持。应用层（如 BAP）负责将这些流同步并分发。
