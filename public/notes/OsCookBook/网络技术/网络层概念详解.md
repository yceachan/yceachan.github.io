# Route &Gateway:

   * Gateway (网关)
        * 它是什么？
          网关是一个网络节点（通常是一个路由器），它连接着两个或多个不同的网络。
        * 它的功能？
          它是你所在网络的进出口。当你的电脑要发送数据给一个不在同一个局域网内的 IP
          地址时，它必须把数据包先发送给网关。
        * Default Gateway (默认网关):
          这是最常见的一种。当你的电脑不知道要把数据包发给谁时（即路由表中没有明确
          的、更精确的规则时），它就会把数据包一股脑地全扔给“默认网关”，让网关来处
          理。这个地址通常是你路由器的 IP 地址（例如 192.168.1.1）。

 - Route (路由)
        * 它是什么？ 路由是路由表 (Routing Table)
          中的一条条目/规则。它不是一个物理设备，而是一条信息。
        * 它的功能？
          这条规则定义了数据包的路径。它回答了这样一个问题：“要去往目标地址
          X，我应该把数据包从哪个网络接口（网卡）发出去，并且交给谁（下一跳地址）？
          ”
        * 路由表的内容：一个路由表里有很多条路由规则。例如：
            * 规则1：要去 192.168.1.0/24 这个网络，直接从 eth0
              网卡发出去就行（这是本地网络）。
            * 规则2：要去 10.0.0.0/8 这个网络，把包交给 IP 地址为 192.168.1.254
              的另一个路由器（这可能是一个特定的路由）。
            * 默认路由 (Default Route)：0.0.0.0/0 或
              default。这条规则表示：“对于上面所有规则都匹配不上的任何其他地址，把
              包交给默认网关 192.168.1.1。”

`Route` (路由) 是一条指令，而 `Gateway` (网关)
  常常是这条指令中涉及的“目的地”或“执行者”。

  看看 ip route 命令的输出，你就能明白一切：

   1 default via 192.168.1.1 dev eth0

   * default: 这是路由 (Route) 规则，适用于所有未知目的地。
   * via 192.168.1.1: 这条规则指示数据包应该通过 192.168.1.1 这个网关
     (Gateway) 来发送。
   * dev eth0: 数据包从 eth0 这个网络接口发出去。
