# 镜像层级：

> 结构1：单镜像（All-in-One Image）——包含 kernel + rootfs + bootloader
>
> sdcard.img
> ├── 分区1：boot（kernel、dtb、bootloader）
> └── 分区2：rootfs（真正的根文件系统）

![image-20251225122040545](https://ali-oss-yceachan.oss-cn-chengdu.aliyuncs.com/img-bed-typora/image-20251225122040545.png)

> 结构2： 多镜像分离（Separate Images）——kernel 和 rootfs 分开烧录
>
> 一些平台（特别是工业板、NXP、TI、Rockchip）会把镜像拆成多个文件：
>
> - `boot.img`（内核 + dtb）
> - `rootfs.img`（根文件系统）
> - `uboot.img`（bootloader）
> - `userdata.img`
> - `vendor.img`

# 烧写工具

## USB烧写工具

芯片厂家一般会提供配套烧写工具；开发板从USB启动，经由USB固件来烧写 img、bootloader、image、dtb

比如 NXP mfgtools ； 100ask 的gui

100ask工具支持单独修改 emmc，Sdcard内部存储，实现在多镜像隔离的层级结构里单独修改kernel，rootfs等。
